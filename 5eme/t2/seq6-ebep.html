💡 Le problème à résoudre

Comment donner des ordres à un robot ou un ordinateur ?

Un robot ne peut rien faire tout seul. Il faut lui donner des instructions.
Ces instructions s'appellent un programme.
Écrire un programme, c'est de la programmation.

Activité 1 — L'algorithme du quotidien

Aide : Un algorithme = une suite d'instructions dans le bon ordre. C'est comme une recette de cuisine !

Remets dans l'ordre les étapes pour faire chauffer de l'eau :

Instruction Ordre (1 à 4)
🔥 Allumer le feu
💧 Remplir la casserole d'eau
⏳ Attendre que l'eau bouille
🍲 Poser la casserole sur le feu

Activité 2 — Les blocs de programmation

Aide : En programmation par blocs (mBlock, Scratch), on empile des blocs comme des Lego :
🟦 Événement = quand ça commence (ex : "Quand drapeau vert cliqué")
🟨 Action = ce que fait le robot (ex : "Avancer")
🟧 Condition = si quelque chose se passe (ex : "Si obstacle détecté")

Associe chaque bloc à son type :

Bloc Événement 🟦 Action 🟨 Condition 🟧
"Avancer de 10 pas"⬜⬜⬜
"Quand drapeau vert cliqué"⬜⬜⬜
"Si obstacle alors..."⬜⬜⬜
"Tourner à droite"⬜⬜⬜

Activité 3 — La boucle

Aide : Une boucle = on répète la même chose plusieurs fois.
Exemple : "Répéter 4 fois : avancer puis tourner" → ça fait un carré !

Complète : Pour que le robot dessine un carré, on répète ........................ fois les instructions "avancer" puis "tourner à droite de 90°".

📝 Ce que je retiens

Choisis le bon mot : programme / algorithme / boucle / condition

Un ........................ est une suite d'instructions dans l'ordre.

Un ........................ est un algorithme écrit pour un ordinateur ou un robot.

Une ........................ permet de répéter des instructions.

Une ........................ permet de faire un choix : "si... alors...".
