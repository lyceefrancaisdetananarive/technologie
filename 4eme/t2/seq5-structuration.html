📌 La programmation par blocs

À retenir
mBlock et Scratch sont des environnements de programmation par blocs visuels. On assemble des blocs comme des pièces de puzzle pour créer un programme, sans écrire de code textuel.
Catégorie de blocsRôleExemples
ÉvénementsDéclenchent le programmeQuand drapeau vert cliqué, quand touche pressée
ContrôleGèrent le flux du programmeRépéter, attendre, si...alors...sinon
ApparenceModifient l'aspect des spritesBasculer sur le costume, afficher message
VariablesStockent des informationsCréer variable, mettre à, ajouter
MessagesCommuniquent entre spritesEnvoyer à tous, quand je reçois

📌 L'interface graphique

À retenir
Une interface graphique est un programme visuel qui permet à l'utilisateur d'interagir avec un système. Dans Scratch, on crée l'interface avec des sprites (objets) et un arrière-plan.
ÉlémentDescription
SpriteObjet graphique programmable (feu, bouton, personnage)
CostumeApparence d'un sprite (ex : feu vert, feu orange, feu rouge = 3 costumes)
Arrière-planImage de fond de la scène (ex : le carrefour)
MessageSignal envoyé entre sprites pour les synchroniser

📌 Synchronisation de systèmes

À retenir
Pour synchroniser plusieurs objets (ex : 2 feux de carrefour), on utilise des messages. Un sprite envoie un message, un autre le reçoit et exécute une action. C'est la communication inter-sprites.
PrincipeApplication au feu tricolore
Les 2 feux ne sont jamais verts en même tempsQuand Feu 1 passe au vert, Feu 2 est rouge (et inversement)
Le message déclenche le changementFeu 1 envoie « phase2 » → Feu 2 reçoit « phase2 » et passe au vert
L'événement piéton interrompt le cycleLe clic sur le bouton met une variable à 1, le cycle vérifie cette variable

📚 Vocabulaire essentiel

Mots clés de la séquence
  • mBlock : logiciel de programmation par blocs compatible avec Arduino, basé sur Scratch
  • Scratch : environnement de programmation visuelle par blocs
  • Sprite : objet graphique programmable dans Scratch/mBlock
  • Costume : apparence d'un sprite (un sprite peut avoir plusieurs costumes)
  • Message : signal envoyé entre sprites pour les synchroniser
  • Variable : espace mémoire nommé qui stocke une valeur modifiable
  • Interface graphique : programme visuel permettant l'interaction homme-machine
  • Synchronisation : coordination de plusieurs éléments pour qu'ils fonctionnent ensemble
