📌 Architecture d'un objet connecté

Un objet connecté (IoT — Internet of Things) est un objet physique capable de communiquer via un réseau.

CoucheRôleExemples
CapteursMesurer une grandeur physiqueTempérature, humidité, mouvement, luminosité
TraitementAnalyser les données captéesMicrocontrôleur (Arduino, ESP32, micro:bit)
CommunicationTransmettre les données via un réseauWi-Fi, Bluetooth, LoRa, Zigbee, 4G/5G
Cloud / ServeurStocker et analyser les données à distanceThingSpeak, Firebase, AWS IoT
Interface utilisateurAfficher les résultats et permettre le contrôleApplication mobile, tableau de bord web
À retenir
Un objet connecté suit le cycle : Capter → Traiter → Communiquer → Stocker → Afficher. Chaque couche a un rôle précis dans la chaîne de l'information.

📌 Protocoles de communication

ProtocolePortéeDébitUsage typique
Bluetooth~10 mMoyenMontres connectées, écouteurs sans fil
Wi-Fi~50 mÉlevéCaméras de surveillance, assistants vocaux
LoRa~15 kmTrès faibleCapteurs agricoles, compteurs intelligents
Zigbee~100 mFaibleDomotique (ampoules, capteurs)
4G / 5GNationaleTrès élevéVoitures connectées, smartphones
À retenir
Le choix du protocole dépend de la portée, du débit et de la consommation d'énergie nécessaires. Un capteur en plein champ utilise LoRa, une caméra HD utilise le Wi-Fi.

📌 Cybersécurité et protection des données

RisqueDescriptionProtection
PiratageUn attaquant prend le contrôle de l'objet à distanceMots de passe forts, mises à jour régulières
EspionnageLes données personnelles sont interceptéesChiffrement des communications (HTTPS, TLS)
Fuite de donnéesLes données stockées sont voléesStockage sécurisé, accès limité
Attaque DDoSDes milliers d'objets piratés saturent un serveurPare-feu, détection d'anomalies
À retenir
Les objets connectés sont vulnérables au piratage. Il faut les protéger par des mots de passe forts, le chiffrement des données et des mises à jour régulières.

📌 RGPD et données personnelles

Le RGPD (Règlement Général sur la Protection des Données) est la loi européenne qui protège les données personnelles :

  • Consentement : l'utilisateur doit accepter explicitement la collecte de ses données
  • Finalité : les données ne peuvent être utilisées que pour le but déclaré
  • Minimisation : ne collecter que les données strictement nécessaires
  • Droit d'accès : l'utilisateur peut consulter ses données à tout moment
  • Droit à l'effacement : l'utilisateur peut demander la suppression de ses données
  • Portabilité : l'utilisateur peut récupérer ses données dans un format standard

Mots clés de la séquence
  • IoT (Internet of Things) : réseau d'objets physiques connectés à Internet
  • Protocole de communication : ensemble de règles pour échanger des données (Wi-Fi, Bluetooth, LoRa)
  • Cloud : serveurs distants qui stockent et traitent les données
  • Cybersécurité : ensemble des pratiques protégeant les systèmes informatiques
  • Chiffrement : transformation des données en code illisible sans la clé
  • RGPD : loi européenne protégeant les données personnelles
  • Domotique : automatisation des équipements de la maison
