Séquences 7-8-9 — Projet Smart Object

Les étapes d'un projet technique

  1. Besoin : identifier le problème à résoudre
  2. Cahier des charges : fonctions de service, contraintes, critères
  3. Conception : composants, schéma, algorithme, organigramme
  4. Fabrication : modélisation 3D, impression, assemblage
  5. Tests : protocole, exécution, débogage, rapport
  6. Présentation : soutenance orale, démonstration

🎓 Révisions DNB — Notions clés de l'année

T1 — Objets connectés et IA

  • Analyse fonctionnelle : fonction de service (QUOI), fonction technique (COMMENT), diagramme bête à cornes
  • Architecture IoT : capteur → microcontrôleur → communication → cloud → application → actionneur
  • IA : embarquée (locale, rapide) vs cloud (puissant, internet) vs hybride
  • Modèle PDA : Percevoir → Décider → Agir
  • Impacts : 4% CO2, fabrication = 70-80% impact, 20% e-déchets recyclés, RGPD

T2 — Systèmes automatisés et programmation

  • Décomposition fonctionnelle : système → sous-systèmes → chaînes fonctionnelles
  • Organigramme : ovale (début/fin), rectangle (action), losange (condition), parallélogramme (E/S)
  • Opérateurs logiques : ET (les deux vraies), OU (au moins une), NON (inverse)
  • Variables : stockent une valeur modifiable
  • Tests : protocole (5 étapes), 4 types d'erreurs (syntaxe, logique, valeur, séquencement)

T3 — Projet et présentation

  • Cahier des charges : besoin, fonctions, contraintes, critères de performance
  • Prototype : modèle d'essai pour valider le concept
  • Fabrication numérique : STL → Slicer → G-code → impression 3D
  • Soutenance : besoin → CdC → conception → fabrication → démo → bilan

📝 Méthode pour l'épreuve du DNB

L'épreuve de Sciences au DNB :
  • Durée : 1h (2 matières parmi Physique-Chimie, SVT, Technologie)
  • Technologie : environ 25 points (sur 50 total)
  • Format : questions sur documents, exercices d'analyse, algorithmes

Conseils de rédaction

  • Lire attentivement les documents avant de répondre
  • Utiliser le vocabulaire technique appris en cours
  • Justifier chaque réponse avec des arguments techniques
  • Pour les algorithmes : écrire en pseudo-code clair avec indentation
  • Pour les organigrammes : utiliser les symboles normalisés
  • Pour les schémas : les légender clairement
