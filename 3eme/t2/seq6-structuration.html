📌 Le protocole de test

Un protocole de test est un document structuré qui décrit comment vérifier qu'un système fonctionne correctement :

ÉlémentDescriptionExemple
Objectif du testCe qu'on veut vérifierVérifier que le capteur détecte un obstacle à 20 cm
Conditions initialesL'état de départ du systèmeRobot allumé, sur surface plane, batterie chargée
ProcédureLes étapes à suivre dans l'ordre1. Placer l'obstacle à 20 cm 2. Lancer le programme 3. Observer
Résultat attenduCe qui doit se passer si tout fonctionneLe robot s'arrête et la LED s'allume
Résultat observéCe qui s'est réellement passéLe robot s'arrête mais la LED ne s'allume pas
VerdictConforme / Non conformeNon conforme → analyser l'erreur
À retenir
Un protocole de test décrit ce qu'on teste, comment, et quel résultat on attend. Il permet de vérifier objectivement si le système respecte le cahier des charges.

📌 Les 4 types d'erreurs en programmation

Type d'erreurQuand ?ExempleComment la trouver ?
Erreur de syntaxeLe programme ne se compile pasPoint-virgule oublié, parenthèse manquanteLe compilateur indique la ligne
Erreur d'exécutionLe programme plante en cours de routeDivision par zéro, dépassement de mémoireMoniteur série, message d'erreur
Erreur logiqueLe programme fonctionne mais donne un mauvais résultatCondition inversée (> au lieu de <)Tests pas à pas, affichage des variables
Erreur de conceptionL'algorithme ne résout pas le bon problèmeL'algorithme de tri ne gère pas les cas limitesRevue de l'algorithme, relecture du CdC
À retenir
Les erreurs de syntaxe sont les plus faciles à corriger (le compilateur les signale). Les erreurs logiques sont les plus sournoises (le programme fonctionne mais mal).

À retenir
Le débogage est une compétence essentielle : afficher les variables avec le moniteur série est la technique la plus simple et la plus efficace pour trouver les erreurs.

📌 Cycle de mise au point

La mise au point d'un programme est un processus itératif :

  1. Écrire le programme (ou une partie)
  2. Compiler → corriger les erreurs de syntaxe
  3. Téléverser et tester
  4. Observer le comportement et comparer au résultat attendu
  5. Déboguer si nécessaire (affichage, isolation)
  6. Améliorer et retourner à l'étape 3

📚 Vocabulaire essentiel

Mots clés de la séquence
  • Protocole de test : document structuré décrivant comment vérifier le fonctionnement d'un système
  • Erreur de syntaxe : erreur dans l'écriture du code empêchant la compilation
  • Erreur logique : erreur dans le raisonnement — le programme fonctionne mais donne un mauvais résultat
  • Débogage : processus de recherche et correction des erreurs dans un programme
  • Moniteur série : outil affichant les données échangées entre l'Arduino et le PC
  • Test unitaire : test d'une seule fonction isolée du reste du programme
  • Itération : cycle répété de développement, test et amélioration
