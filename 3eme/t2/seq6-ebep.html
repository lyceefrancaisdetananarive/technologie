💡 Le problème à résoudre

Comment vérifier qu'un programme fonctionne correctement ?

Quand on écrit un programme, il y a souvent des erreurs (bugs).
Pour les trouver, on fait des tests.
Pour les corriger, on fait du débogage.

Activité 1 — Les types d'erreurs

Aide : Il y a 3 types d'erreurs principaux :
🔴 Erreur de syntaxe = faute d'écriture, le programme ne démarre pas
🟡 Erreur d'exécution = le programme plante en cours de route
🟢 Erreur logique = le programme marche, mais fait la mauvaise chose

Classe chaque erreur :

ErreurSyntaxe 🔴Logique 🟢
J'ai oublié un point-virgule⬜⬜
La LED s'allume quand elle devrait s'éteindre⬜⬜
J'ai écrit « if » au lieu de « IF » (mauvaise casse)⬜⬜
Le robot tourne à gauche au lieu de tourner à droite⬜⬜

Activité 2 — Ma fiche de test

Aide : Un protocole de test dit : ce qu'on teste, comment, et ce qu'on attend.

Remplis ta fiche de test :

Ce que je teste
Comment je teste
Résultat attendu
Résultat observé
Conforme ?⬜ Oui     ⬜ Non

📝 Ce que je retiens

Choisis le bon mot : test / débogage / syntaxe / logique

Un ........................ vérifie si le programme fonctionne comme prévu.

Le ........................ = chercher et corriger les erreurs.

Une erreur de ........................ empêche le programme de démarrer.

Une erreur ........................ = le programme marche mais fait la mauvaise chose.
