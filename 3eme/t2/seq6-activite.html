💡 Situation déclenchante

Tu as programmé un mBot pour suivre une ligne, mais il zigzague et perd la ligne dans les virages. Comment identifier le problème et le corriger de manière méthodique ?

Problématique : Comment tester, valider et corriger un prototype de manière méthodique et structurée ?

Activité 1 — Protocole de test

1

Planifier les tests avant de les exécuter

Rédige un protocole de test pour valider le comportement du mBot.

Les 5 étapes d'un protocole de test :
  1. Définir les critères de réussite (issu du cahier des charges)
  2. Planifier les scénarios de test
  3. Exécuter les tests dans des conditions contrôlées
  4. Observer et noter les résultats
  5. Analyser les écarts entre attendu et observé

Activité 2 — Débogage méthodique

2

Identifier et corriger les erreurs

Applique une méthode de débogage structurée pour corriger le programme du mBot.

Les 4 types d'erreurs en programmation :
  • Erreur de syntaxe : blocs mal connectés, paramètres manquants
  • Erreur de logique : algorithme incorrect (ex : > au lieu de <)
  • Erreur de valeur : seuils incorrects (vitesse, distance)
  • Erreur de séquencement : instructions dans le mauvais ordre

a) Pour chaque test échoué, identifie le type d'erreur et propose une correction :
