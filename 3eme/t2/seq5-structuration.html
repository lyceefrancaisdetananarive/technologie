📌 Variables et types de données

TypeContenuExemple Arduino
intNombre entierint compteur = 0;
floatNombre décimalfloat temperature = 23.5;
boolVrai ou Fauxbool estAllume = true;
StringTexteString message = "Bonjour";
À retenir
Chaque variable a un type qui détermine la nature des données qu'elle peut stocker. Le type doit être choisi en fonction de l'usage (entier pour un compteur, float pour une mesure).

📌 Opérateurs logiques

OpérateurSymboleSignificationExemple
ET (AND)&&Vrai si les DEUX conditions sont vraiesif (temp > 30 && humidite > 80)
OU (OR)||Vrai si AU MOINS UNE condition est vraieif (boutonA || boutonB)
NON (NOT)!Inverse la valeur (vrai → faux, faux → vrai)if (!capteurPresence)
À retenir
Les opérateurs logiques permettent de combiner plusieurs conditions : ET (les deux vraies), OU (au moins une vraie), NON (inverse).

📌 Tables de vérité

Les tables de vérité montrent tous les résultats possibles d'une opération logique :

ABA ET BA OU BNON A
FAUXFAUXFAUXFAUXVRAI
FAUXVRAIFAUXVRAIVRAI
VRAIFAUXFAUXVRAIFAUX
VRAIVRAIVRAIVRAIFAUX
À retenir
La table de vérité liste toutes les combinaisons possibles des entrées et le résultat correspondant. C'est un outil indispensable pour vérifier la logique d'un programme.

📌 Fonctions et modularité

Les fonctions permettent de découper un programme en blocs réutilisables :

ConceptExplicationExemple
Définir une fonctionCréer un bloc de code nommévoid clignoter() { ... }
Appeler une fonctionExécuter le bloc quand on en a besoinclignoter();
ParamètreValeur passée à la fonction pour la personnaliservoid clignoter(int duree)
Valeur de retourRésultat renvoyé par la fonctionint lireDistance() { return dist; }
À retenir
Les fonctions rendent le programme modulaire (découpé en blocs), lisible (chaque bloc a un nom clair) et réutilisable (on peut appeler la même fonction plusieurs fois).

📚 Vocabulaire essentiel

Mots clés de la séquence
  • Type de variable : nature des données stockées (int, float, bool, String)
  • Opérateur logique : ET, OU, NON — pour combiner des conditions
  • Table de vérité : tableau listant toutes les combinaisons d'entrées et résultats
  • Fonction : bloc de code nommé, réutilisable et paramétrable
  • Paramètre : valeur passée à une fonction pour personnaliser son comportement
  • Modularité : organisation d'un programme en blocs indépendants
