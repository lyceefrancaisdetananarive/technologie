📌 Analyse fonctionnelle d'un objet connecté

L'analyse fonctionnelle décrit les services rendus par un objet technique à son environnement.

OutilCe qu'il montreÉléments
Diagramme bête à cornesLe besoin fondamentalÀ qui rend-il service ? Sur quoi agit-il ? Dans quel but ?
Diagramme pieuvreLes fonctions de serviceL'objet au centre, les éléments de l'environnement autour, reliés par des fonctions
Cahier des chargesLes exigences détailléesFonctions, critères, niveaux, flexibilité
À retenir
Le diagramme pieuvre place l'objet au centre et montre ses relations avec les éléments de l'environnement. Les FP (fonctions principales) relient deux éléments extérieurs, les FC (fonctions contraintes) relient l'objet à un seul élément.

📌 Architecture d'un objet connecté

CoucheRôleExemples
PerceptionCapter des données du monde physiqueCapteurs (température, mouvement, GPS)
Traitement localAnalyser et filtrer les donnéesMicrocontrôleur (ESP32, Arduino)
CommunicationTransmettre les donnéesWi-Fi, Bluetooth, LoRa, 5G
Cloud / ServeurStocker, analyser, croiser les donnéesAWS, Google Cloud, ThingSpeak
ApplicationPrésenter les résultats et permettre le contrôleApp mobile, dashboard web
À retenir
Un objet connecté suit le cycle : Capter → Traiter → Communiquer → Stocker → Afficher/Agir. Les flux de données circulent entre chaque couche.

📚 Vocabulaire essentiel

Mots clés de la séquence
  • Analyse fonctionnelle : méthode décrivant les services rendus par un objet à son environnement
  • Diagramme pieuvre : représentation graphique des fonctions de service d'un produit
  • FP (Fonction Principale) : service principal rendu par l'objet, reliant deux éléments extérieurs
  • FC (Fonction Contrainte) : exigence imposée par un élément de l'environnement
  • Flux de données : circulation des informations entre les composants d'un système
  • MQTT : protocole de communication léger pour les objets connectés
  • Cloud : serveurs distants pour le stockage et le traitement des données
